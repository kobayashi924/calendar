# 家族カレンダーをGitHub Pagesで公開する手順

この一式をGitHubに置くと、`source/` のテキストを編集するだけで、
家族用カレンダーが自動で生成・公開されます。Macのフォルダもスクリプトも不要になります。

公開後のURL（例）: `https://ユーザー名.github.io/family-calendar/`

---

## 用意するもの
- GitHubアカウント（無料）→ https://github.com/signup
- このフォルダ一式（`build.py` / `.github/workflows/deploy.yml` / `source/`）

---

## 手順

### 1. リポジトリを作る
1. GitHubにログイン → 右上「＋」→ **New repository**
2. Repository name に `family-calendar` などを入力
3. **Public** を選択（Pagesの無料公開はPublicが簡単）
4. 「Create repository」

### 2. ファイルをアップロードする
1. 作成直後のページで「**uploading an existing file**」をクリック
2. このフォルダの中身を**まとめてドラッグ＆ドロップ**
   - `build.py`
   - `source` フォルダ（中の6つの .txt ごと）
   - ※ `.github` フォルダは隠しフォルダのため、ドラッグでうまく入らない場合は手順3で作成
3. 下の「Commit changes」

### 3. ワークフローを作る（.github がアップロードできなかった場合のみ）
1. 「Add file」→「**Create new file**」
2. ファイル名の欄に次を**そのまま入力**（スラッシュでフォルダが作られます）:
   `.github/workflows/deploy.yml`
3. 同梱の `deploy.yml` の中身をコピーして貼り付け
4. 「Commit changes」

### 4. Pagesを有効にする
1. リポジトリの **Settings** → 左メニュー **Pages**
2. 「Build and deployment」→ Source を **GitHub Actions** に設定
3. 完了。Settings → Pages にURLが表示されます

### 5. 動作確認
- リポジトリ上部の **Actions** タブで、緑のチェックが付けば公開成功
- 表示されたURLを開いてカレンダーを確認 → 家族にそのURLを共有

---

## 予定を更新するとき（あなただけ）
1. GitHubでリポジトリを開く → `source` フォルダ → 編集したい人の `.txt`
2. 右上の **鉛筆アイコン** をクリックして編集
3. 書式はこれまで通り「`5月3日 熊の湯`」のように1行1予定
   - `[ ]` で囲むと小さな補足メモになります（例: `6月13日 [DIY？]`）
4. 「Commit changes」→ 約1分で公開ページが自動更新されます
   - スマホのGitHubアプリ／ブラウザからでも編集できます

## 年度を切り替えるとき
- `build.py` の先頭 `START_YEAR = 2026` の数字を変更してCommit

## バッジの対応
| 記号 | 色 | 人 |
|---|---|---|
| な | オレンジ | なる実 |
| や | 紺 | 弥生 |
| ゆ | 緑 | 由布子 |
| み | 赤紫 | 充希 |

「全員」の予定はバッジなしで全幅表示されます。
